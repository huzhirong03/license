## GitHub 部署步骤

### 第1步：创建 GitHub 仓库

1. 打开 https://github.com
2. 登录你的账号
3. 点击右上角 **+** → **New repository**
4. 填写信息：
   - Repository name: `license-generator`（或其他名字）
   - Description: `激活码生成器`
   - 选择 **Public**（必须公开才能用 GitHub Pages）
   - 勾选 **Add a README file**
5. 点击 **Create repository**

### 第2步：上传文件

**方法A：网页上传（简单）**

1. 进入刚创建的仓库
2. 点击 **Add file** → **Upload files**
3. 把以下文件/文件夹拖入：
   - `index.html`
   - `css/` 文件夹
   - `js/` 文件夹
4. 在底部填写 Commit message: `上传激活码生成器`
5. 点击 **Commit changes**

**方法B：Git 命令行（推荐）**

```bash
# 克隆仓库
git clone https://github.com/你的用户名/license-generator.git

# 进入目录
cd license-generator

# 复制文件（把文件复制到这个目录）
# index.html, css/, js/

# 添加所有文件
git add .

# 提交
git commit -m "上传激活码生成器"

# 推送
git push origin main
```

### 第3步：开启 GitHub Pages

1. 进入仓库页面
2. 点击 **Settings**（设置）
3. 左侧菜单找到 **Pages**
4. Source 选择：
   - Branch: **main**
   - Folder: **/ (root)**
5. 点击 **Save**
6. 等待几分钟，页面会显示网址

### 第4步：获取访问网址

部署成功后，网址格式为：
```
https://你的用户名.github.io/license-generator/
```

例如：
```
https://fage.github.io/license-generator/
```

## 添加新程序

当你开发了新程序需要激活时：

### 1. 修改 js/generator.js

找到 `PROGRAMS` 对象，添加新程序：

```javascript
const PROGRAMS = {
    'R2V': {
        name: '矢量转换工具',
        enabled: true
    },
    'VBA_TOOL': {
        name: '宏嫖边工具',
        enabled: true  // 改成 true
    },
    // 添加新程序
    'NEW_APP': {
        name: '新程序名称',
        enabled: true
    }
};
```

### 2. 修改 index.html

找到程序选择下拉框，添加选项：

```html
<select id="programSelect">
    <option value="R2V">矢量转换工具</option>
    <option value="VBA_TOOL">宏嫖边工具</option>
    <option value="NEW_APP">新程序名称</option>
</select>
```

### 3. 在新程序中实现验证

新程序需要实现相同的激活码验证算法：
- 输入：机器码 + 类型代码 + SECRET_KEY
- 算法：SHA256 + MD5
- 输出格式：XXXX-XXXX-XXXX-TTTT

## 常见问题

### Q: 为什么网页打不开？

1. 检查 GitHub Pages 是否开启
2. 等待几分钟让部署生效
3. 确认网址是否正确

### Q: 生成的激活码和 Python/C# 版本一样吗？

是的！算法完全相同：
- SHA256(机器码 + 类型代码 + SECRET_KEY)
- 同样的输入 = 同样的输出

### Q: 密码安全吗？

密码存储在 JavaScript 代码中，有一定风险。但因为：
1. 网址不公开
2. 只有你自己知道网址
3. 即使看到密码，也需要知道 SECRET_KEY 才能生成有效激活码

所以实际风险较低。

### Q: 如何更新代码？

1. 修改本地文件
2. 重新上传到 GitHub（覆盖原文件）
3. 或使用 git push

### Q: 手机上能用吗？

可以！网页是响应式设计，手机浏览器直接访问网址即可。

## 技术细节

### 激活码生成算法

```
1. 输入：机器码 + 类型代码 + SECRET_KEY
2. 计算 SHA256 哈希
3. 取哈希的前12位
4. 计算类型代码的 MD5，取前4位
5. 格式化：XXXX-XXXX-XXXX-TTTT
```

### SECRET_KEY

```
fage_laser_2024_secret_key_do_not_share
```

⚠️ 这个密钥必须和所有程序中的密钥一致！
